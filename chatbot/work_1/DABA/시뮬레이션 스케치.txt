<카카오톡에서 해야될일>
1. 인사

2. 설정할 등산로와 시간을 입력하라는 안내

3. 등산로 입력을 받아옴 
    --> 자연어모델(의도: 조회, 개체명 인식: 장소)
	- 강원도의 땡땡1로 찾아줘 --> (의도: 조회, 개체명 인식: 장소) --> 출력: 강원, 땡땡2로 등산로를 조회합니다 
    --> 답변검색(DB에서 유사 데이터 조회)
	- 임베딩, 코사인 유사도
    --> 유사 데이터 출력
    --> 사용자 네, 아니오(등산로 리스트 반환하여 사용자가 선택하도록)

4. 등산시작 시간 입력 받아옴
    --> 자연어모델(의도: 조회, 개체명 인식: 시각)
	- 년 월 일시 --> (의도: 조회, 개체명 인식: 시간) --> 출력: 사용자 입력 시간을 조회합니다
    --> 답변검색(시각 식별 함수 필터링: 년월일시 체크, Datetime으로 변환 가능 체크)
    --> 조회 가능여부 출력(실패시 입력 포맷 재안내)

(입력 완료시, 모델 예측 수행 및 3가지 정보 반환)

5. 반환 정보 안내 메세지


<서비스 시뮬레이션>

- 큰그림
1. 고객의 등산로 셋팅(등산로, 시작시각)
2. [등산로, 등산시작 시간] 값에 대한 등산로, 예측바람장 출력정보 반환
3. [등산로, 등산시작 시간] 값에 대한 위험바람구간 출력정보 반환

- 상세
1. 고객의 등산로 셋팅
  1.1. 등산로 설정
    (방법) 
            1. 지역 or 등산로명 입력받음
            2. DB에서 조회 및 반환
            3. 검색 안되면 지역내 등산로 리스트 반환
            4. 사용자 설정 등산로 정보 받아옴
  1.2. 등산 시작시간 설정
    (방법)
            1. "등산로 설정" 후, 등산의 시작시간 입력 받음(예측 기준시간 설정하기 위해)
               1.1 년월일시(KST)

2. [등산로, 등산시작 시간] 값에 대한 출력정보 반환
  2.1 to_client 함수
   2.1.1 DB 연결
   2.1.2 고객 입력정보 json으로 받아옴
   2.1.3 편차보정모델 수행(입력: 등산로,시작시각) --> 보정된 예측바람장 반환
   2.1.4 등산로 500m 간격으로 예측바람장 내삽
   2.1.5 경고알람 모델 수행(입력: 등산로, 예측바람장) --> 위험지역 존재시 경고 문자 반환
   2.1.6 3개 반환(보정된 예측바람장, 등산로 예측바람장, 경고문자)

3. 안내서비스 제출
  3.1 이미지 파일 제출(위성지도 + 등산로 + 예측바람장)
  3.2 경고문자 제출
